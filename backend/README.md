# Fish Weight Prediction: Centralized Deployment

This folder contains the standalone, centralized deployment of the fish weight prediction model. It uses the best models trained via the Federated Learning (FL) pipeline but operates entirely independently without any FL infrastructure or network communication.

## 📂 Required Files and Folder Structure

To run the inference script, ensure your folder structure looks exactly like this:

```text
centralized-deployment/
│
├── README.md                  <-- This file
├── source-code/
│   ├── inference.py           <-- The main standalone inference script
│   ├── model.py               <-- Defines the UNet and BPNN architectures
│   └── clients_utils.py       <-- Contains feature extraction math (e.g., area, perimeter)
│
└── models/                    <-- Your final trained intelligence
    ├── global_unet.pth        <-- The trained UNet segmentation weights
    ├── global_bpnn.pth        <-- The trained BPNN regression weights
    └── global_scaler.pkl      <-- The scaler used to normalize morphological features
```

---

## ⚙️ The Inference Pipeline

The script `inference.py` acts as a complete, self-contained AI pipeline. When you run `predictor.predict("image.jpg")`, the following steps happen in real-time:

1. **Semantic Segmentation (UNet):**
   - The script loads the raw RGB image and resizes it.
   - The UNet model processes the image and outputs a binary 2D mask (white for the fish body, black for the background).
2. **Feature Extraction:**
   - Computer vision tools calculate physical characteristics of the mask (Area, Perimeter, Major Axis Length, Minor Axis Length).
   - The pixel measurements are converted to centimeters using your specific camera calibration (`cm_per_pixel`).
3. **Feature Scaling:**
   - The extracted features are scaled between 0 and 1 using the `global_scaler.pkl` to match the format the BPNN was trained on.
4. **Mass Regression (BPNN):**
   - The scaled features are fed into the BPNN model.
   - The BPNN outputs a single, continuous float value representing the final predicted weight.

---

## 📥 Inputs

* **Input Image:** A raw `.jpg`, `.png`, or `.jpeg` image of a fish taken under the same camera setup and angle as your training data.
* **Camera Calibration (`cm_per_pixel`):** Passed into the `FishWeightPredictor` initialization. This tells the script how big a single pixel is in the real world. (Default is `0.038`). If your camera moves closer or further away, this value **must** be updated.

## 📤 Outputs

The script returns a Python dictionary containing:

1. **`predicted_weight_g` (float):** The final predicted weight in grams, rounded to 2 decimal places. (e.g., `1245.50`)
2. **`features` (dict):** A dictionary of the raw physical measurements (Area, Perimeter, etc.) extracted from the image. Useful for logging and debugging.
3. **`mask` (numpy array):** The 2D binary segmentation mask generated by the UNet. You can save this as an image to verify that the UNet properly identified the fish and didn't include the background.

---

## 🚀 How to Run

Simply execute the script from your terminal:

```bash
python source-code/inference.py
```

Ensure you update the `test_image` path at the bottom of `inference.py` to point to a valid fish image before running!

---

## 🛠️ Generating the Scaler (Optional)

The `global_scaler.pkl` file normalizes the physical characteristics (area, perimeter, etc.) so the neural network can process them correctly. This file was generated exactly once using all the fish images in the training dataset.

**You do NOT need to generate this file for normal inference.** You only need to regenerate the scaler if you add new training data or change the physical features being extracted.

If you need to regenerate the scaler, you can use the `generate_global_scaler.py` script located in the `fl-dp-sa/scripts/` folder of the main repository:

```bash
cd ../fl-dp-sa
python scripts/generate_global_scaler.py
```

This will automatically extract the features from all training masks, calculate the global mean and variance, and overwrite the `global_scaler.pkl` file with the new normalization parameters.
